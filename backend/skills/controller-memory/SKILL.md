# Controller Memory

name: controller-memory
description: 主控模型用于维护长期记忆（文件型记忆）的技能。通过此技能，你可以将关于用户的稳定偏好、重要事实和项目上下文记录到长期记忆中，避免被遗忘。

## 核心理念

你拥有一个名为 `controller_memory` 的长期记忆库（最多 200 行）。
当你在对话中发现用户有明确的偏好、长期的项目事实或重要的上下文时，应该主动使用此技能进行记录。

## 记忆分级与类型

记忆包含 `priority` (优先级) 和 `kind` (类型)。
当记忆达到 200 行上限时，系统会优先淘汰低优先级的旧记忆。

### 优先级 (priority)
- **P1**: 最高优先级。用于用户稳定的个人偏好、性格特点、核心原则等。**几乎不会被淘汰**。
- **P2**: 中等优先级。用于阶段性的长期事实、重要项目背景等。
- **P3**: 最低优先级。用于近期的临时上下文、普通事实等。最容易被淘汰。

### 类型 (kind)
- `preference`: 用户偏好（通常搭配 P1）
- `fact`: 客观事实（通常搭配 P2/P3）
- `project`: 项目信息（通常搭配 P2）
- `context`: 临时上下文（通常搭配 P3）

## 使用指南

1. **主动记录**: 当用户表达了明确的偏好（例如“我更喜欢看整理后的学习要点，不想直接啃源码”），你应该立刻调用 `remember_important_fact` 将其记录为 `P1` + `preference`。
2. **避免重复**: 系统会自动根据文本内容去重，但你应尽量保持记录的简洁和独立。
3. **维护清理**: 你可以通过 `list_controller_memory` 查看当前记录，并通过 `delete_memory_fact` 删除已过时或不再适用的记忆。
4. **升级重要事实**: 如果你发现某条 P3 的事实变得非常重要，可以使用 `pin_memory_fact` 将其升级为 P1。

请在适当的时候，主动调用这些工具，以提供更个性化、更连贯的服务。
